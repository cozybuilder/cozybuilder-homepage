# PROGRAM OPERATING MODEL — CozyBuilder 홈페이지·프로그램 운영 구조

## 1. 홈페이지의 역할

CozyBuilder 홈페이지는 단순 회사 소개 사이트가 아니라 다음 기능을 담당하는 중앙 플랫폼이다.

- CozyBuilder 브랜드 허브
- 공통 회원가입·로그인
- 무료 프로그램 유입
- 프로그램 탐색·소개
- 마이페이지의 내 프로그램 통합 조회
- 무료 이용·무료 체험·구독·구매 상태 표시
- 유료·독립 서비스의 실행 진입점
- 프로그램별 이용·전환 분석

홈페이지 자체의 입장료나 일괄 구독을 기본 수익모델로 삼지 않는다. 사용자는 필요한 유료 프로그램에만 개별 결제한다.

## 2. 두 가지 운영 모델

### A형 — 홈페이지 귀속형

무료이거나 작고 단순한 프로그램을 홈페이지 내부에 귀속한다.

적합 조건:

- 무료 제공이 핵심인 프로그램
- 회원 유입과 재방문을 만드는 미끼 상품
- 독립 브랜드가 필요하지 않음
- 별도 서버·고비용 AI API·복잡한 결제가 없음
- 홈페이지 저장소에서 빠르게 구현·운영 가능
- 별도 배포보다 홈페이지 내부 편입이 효율적

구조:

- URL: `cozybuilder.co.kr/programs/...` 또는 홈페이지 내부 앱 경로
- 홈페이지 인증·세션을 그대로 사용
- 마이페이지의 `내 프로그램`에 자동 표시
- 홈페이지 분석·권한 체계를 그대로 사용
- 무료 이용 상태로 관리

목적:

무료 프로그램 사용을 위해 사용자가 CozyBuilder 계정을 만들고, 마이페이지에서 다른 무료·유료 프로그램을 발견하도록 한다.

### B형 — 독립 서비스형

유료 SaaS, 자체 데이터와 운영비가 있는 프로그램, 독립 성장 가능성이 있는 서비스를 별도 프로젝트로 운영한다.

적합 조건:

- 자체 상품명·고객·요금제가 있음
- 결제·구독·AI API·파일 업로드·비동기 작업이 있음
- 민감하거나 대량의 업무 데이터를 저장함
- 홈페이지와 독립적으로 자주 개발·배포해야 함
- 독립 브랜드·도메인·매각 가능성이 있음
- 홈페이지 저장소에 포함하면 복잡도와 장애 영향이 커짐

구조:

- 별도 GitHub 저장소
- 별도 배포와 서브도메인 또는 독립 도메인
- CozyBuilder 공통 계정의 사용자 ID를 사용
- 서비스 업무 데이터는 서비스별로 분리
- 홈페이지 마이페이지에서 이용 상태와 실행 버튼 제공
- 프로그램별 개별 결제·구독

홈페이지는 서비스 본체를 품지 않고 소개·로그인·권한·구독 상태·실행 연결을 담당한다.

## 3. 마이페이지 상태 모델

마이페이지의 상위 명칭은 `내 프로그램`으로 한다.

표시 상태:

- 무료 이용 중
- 무료 체험 중
- 구독 중
- 구매 완료
- 이용 종료

A형과 B형 모두 같은 목록에 표시한다. B형의 실행 버튼은 외부 서비스 URL로 이동한다.

## 4. 공통 로그인 원칙

- 프로그램마다 별도 회원 DB를 만들지 않는다.
- 홈페이지가 사용하는 공통 인증 공급자와 사용자 ID를 재사용한다.
- 각 독립 서비스는 필요한 콜백 URL과 인증 클라이언트 설정만 추가한다.
- 완전한 서브도메인 간 SSO는 필요 시 중앙 인증 리디렉션 또는 공통 세션 구조로 확장한다.
- 통합 대상은 사용자 신원과 이용권이며, 서비스 업무 데이터는 분리한다.

## 5. 결제·구독 원칙

- 홈페이지 회원가입은 무료다.
- 홈페이지 전체 이용을 조건으로 한 강제 구독은 두지 않는다.
- 유료 프로그램은 프로그램별 요금제로 결제한다.
- 여러 프로그램을 묶는 통합 구독은 충분한 유료 상품과 실제 수요가 생긴 뒤 검토한다.
- 마이페이지에서는 모든 프로그램의 이용권·구독 상태를 통합 조회한다.

## 6. 신규 프로그램 판정 절차

신규 프로그램 시작 전 아래를 판단한다.

1. 무료 유입 상품인가, 독립 유료 상품인가?
2. 별도 운영비·AI API·결제·민감 데이터가 있는가?
3. 독립 배포와 잦은 업데이트가 필요한가?
4. 독립 브랜드와 URL이 유리한가?
5. 홈페이지 저장소에 포함할 경우 복잡도가 과도하게 증가하는가?

판정:

- 무료·소형·저비용·홈페이지 유입 목적이면 A형
- 유료·대형·독립 운영·별도 데이터가 필요하면 B형
- 애매하면 MVP는 A형으로 시작할 수 있으나, 분리 비용이 예상되면 처음부터 B형으로 한다.

## 7. 현재 적용

### A형 유지 후보

- 홈페이지 내부의 기존 무료 프로그램
- 전자책 만들기
- ClipMiner 웹 기능

기존 프로그램은 즉시 이전하거나 폐기하지 않는다. 현재 구조를 보존하고 실제 사용 반응을 기준으로 추가 개발 여부를 판단한다.

### B형 기본 적용 후보

- 코지임대
- 평생사진관
- AI Promotion
- 향후 유료 SaaS와 독립 서비스

## 8. 운영 원칙 요약

> 무료 프로그램은 홈페이지에 귀속해 회원 유입과 교차 노출에 사용한다. 유료·대형 서비스는 독립 배포하되, 공통 로그인과 마이페이지를 통해 하나의 CozyBuilder 생태계로 연결한다.
